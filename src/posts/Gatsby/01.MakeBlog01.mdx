## 정적 웹사이트 생성기
개인블로그를 만들기 위해 검색을 해보다 정적웹사이트 생성기인 gatsby를 알게 되었다.  
정적 웹사이트 생성기(SSG, Static Site Generator)는  
HTML, CSS, JavaScript로 구성된 정적 페이지 사이트를 만들어주는 도구이다.
<a
  href="https://blog.outsider.ne.kr/1426"
  target="SSG"
>
  [정적 사이트 생성기 Gatsby]
</a>
블로그의 글을 읽으며 어느정도 개념을 파악할 수 있었다.  <br />
Jekyll, Hugo, Nuxt.js 등 여러가지 정적 웹사이트 생성기가 있지만  <br />
React 컴포넌트를 사용할 수 있는 Gatsby를 사용하기로 결정했다.

## gatsby 설치
  - 홈 디렉토리 이동 후 프로젝트 디렉토리 생성.
  ```shell
    cd ~
    mkdir WebstormProjects
    cd WebstormProjects
  ```
<br />
  
## npm 설치
  - gatsby-cli global 설치
  ```shell
    npm install -g gatsby-cli
  ```
<br />

## 신규 gatsby 프로젝트 생성
- <a href="https://www.gatsbyjs.com/starters?" target="gatsby_starter">[gatsby starter]</a>에서 맘에드는 starter(템플릿)를 설치할 수 있다.
- 따로 starter를 지정하지 않을 경우에는 gatsby-starter-default로 만들어진다. 
  ```shell
    gatsby new jpark6.github.io
  ```
<br />

## gatsby 프로젝트 github 연결
  - 로컬에 생성한 gatsby 프로젝트를 github repository에 push. 
  ```shell
    git init
    git commit -m "commit message"
    git branch -M main
    git remote add origin https://github.com/jpark6/jpark6.github.io
    git push -u origin main
  ```
<br />

## gatsby 배포
  - 배포를 위해 gh-pages 브랜치를 만든다.  
  - main branch에 배포를 할경우, 소스코드가 날라가고 빌드된 파일만 남기때문에 배포용 브랜치를 따로 만든다.  
  ```shell
    git branch gh-pages
  ```
<br />
  - package.json에 배포 명령어 추가
  ```shell
    "scripts": {
      ...
      "deploy": "gatsby build && gh-pages -d public -b gh-pages"
    },
  ```
  - 배포 명령어 실행.
  ```shell
    npm run deploy
  ```
